• INSIGHT •

Code Architecture


Shibanobi's Revenge is the first game I made that had levels. In order to facilitate level creation in the engine, I created a seamless way for level screens to transition between each other. This way, the player doesn't have to think about camera control and can see everything he needs to see on a single screen.

Every single level follows a pattern. Each screen is numbered and wrapped inside a box collider. This way, the camera controller knows the boundaries of the current screen and knows which portion of the level it has to transition to next.

Shibanobi's Revenge is a game where the player has to perform precise movements in order to reach the end of each level. In order to help the player achieve this, every movement ability is cancellable by another movement ability. Therefore, I used a Finite State Machine to make every state the player can be in, and give them the ability to switch between them when they need to.

Level Design


The level design was a very important portion of the game. It lets players learn the game's mechanics and apply them when the game challenges their skill. Every level was first drawn on paper using a grid. In order to have a seamless gameplay, we wrote down the distance of every ability and used that information to make the obstacles.
